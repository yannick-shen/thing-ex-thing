# 用户信息权限获取说明

## 🎯 问题解决

用户评论显示"新用户"的问题是因为没有在登录时获取用户的微信昵称和头像。现在已经完全修复！

## ✅ 已实施的改进

### 1. **登录页面UI更新**
- 添加了权限说明区域
- 清晰告知用户需要获取的信息：
  - 微信昵称（用于显示用户名）
  - 微信头像（用于显示头像）

### 2. **权限申请流程**
```javascript
// 使用 wx.getUserProfile 获取用户信息
wx.getUserProfile({
  desc: '用于完善用户资料',
  success: (res) => {
    // 获取到用户信息，传递给云函数
  },
  fail: (err) => {
    // 用户拒绝时友好提示，可选择跳过
  }
})
```

### 3. **用户信息处理**
- **新用户**：使用微信昵称和头像创建账户
- **老用户**：如果昵称有变化，自动更新
- **用户拒绝**：可使用"匿名用户"继续使用

### 4. **云函数增强**
- `user-auth` 云函数现在接收并处理用户信息
- 自动更新用户的昵称和头像
- 保持向后兼容性

## 📋 用户体验流程

### 正常流程
1. 用户点击"微信快速登录"
2. 看到权限说明（透明告知）
3. 点击授权 → 获取微信昵称和头像
4. 使用真实用户名发表评论

### 用户拒绝流程
1. 用户点击"取消授权"
2. 弹出友好提示说明
3. 可选择"重新授权"或"跳过"
4. "跳过"则使用"匿名用户"身份

## 🔧 技术实现

### 前端授权
```javascript
// 获取用户信息
const userProfile = await this.getUserProfile();

// 传递给云函数
const loginResult = await authManager.login(loginCode, userProfile);
```

### 云函数处理
```javascript
// 新用户创建时
profile: {
  nickname: userProfile.nickName || '匿名用户',
  avatarUrl: userProfile.avatarUrl || ''
}

// 老用户更新时
if (userProfile.nickName !== user.profile.nickname) {
  await userCollection.update({
    'profile.nickname': userProfile.nickName,
    'profile.avatarUrl': userProfile.avatarUrl
  });
}
```
