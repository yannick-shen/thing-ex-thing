# 用户信息调试指南

## 🐛 问题描述
用户名显示为"微信用户"而不是真实微信昵称，头像也没有更新。

## 🔍 可能原因分析

### 1. **API 兼容性问题**
- `wx.getUserProfile` 可能在某些环境下不可用
- 微信小程序基础库版本问题
- 用户授权状态异常

### 2. **数据传递问题**
- 前端获取到但未正确传递给云函数
- 云函数接收到但处理逻辑有问题
- 数据库更新失败

## ✅ 已实施的增强调试

### 1. **双重API尝试**
```javascript
// 优先尝试 wx.getUserInfo
wx.getUserInfo({
  success: (res) => {
    // 如果已授权，直接使用
  },
  fail: (err) => {
    // 失败后尝试 wx.getUserProfile
    wx.getUserProfile({...})
  }
})
```

### 2. **详细日志记录**
- 前端：每一步获取都有日志
- 云函数：接收到的用户信息详细记录
- 数据库：更新操作的具体结果

### 3. **更强的数据验证**
```javascript
// 确保昵称不为空
nickname: (userProfile && userProfile.nickName && userProfile.nickName.trim()) ? userProfile.nickName.trim() : '新用户'

// 检查是否需要更新
if (userProfile.nickName !== (user.profile && user.profile.nickname)) {
  // 执行更新
}
```

## 🔧 调试步骤

### 第一步：重新部署并测试
请重新部署以下文件：
1. **前端页面**：`pages/login/login.js`
2. **云函数**：`cloudfunctions/user-auth/index.js`
3. **工具类**：`utils/auth.js`

### 第二步：检查前端日志
在微信开发者工具控制台查看：

```
获取用户信息成功: {userInfo: {nickName: "真实昵称", avatarUrl: "头像URL"}}
```

如果看到：
- `{userInfo: {nickName: "微信用户", avatarUrl: ""}}`
- 则说明API调用有问题

### 第三步：检查云函数日志
在云开发控制台查看 `user-auth` 云函数日志：

```
✅ userProfile: {nickName: "真实昵称", avatarUrl: "头像URL"}
✅ userProfile类型: object
✅ userProfile.nickName: 真实昵称
✅ userProfile.avatarUrl: 头像URL
```

如果看到：
- `userProfile: null` 或 `undefined`
- `nickName: "微信用户"`
- 则说明数据传递有问题

## 🎯 根据日志结果判断

### 情况A：前端获取到正确信息
如果前端日志显示正确的昵称和头像：
- ✅ 问题出在云函数处理
- 检查云函数日志确认数据传递
- 查看数据库更新结果

### 情况B：前端获取到错误信息
如果前端日志显示"微信用户"：
- ❌ API调用有问题
- 可能需要使用其他方式获取用户信息
- 检查小程序基础库版本

### 情况C：数据传递中断
如果前端正确但云函数接收错误：
- ❌ 传递过程中有问题
- 检查 `authManager.login` 调用
- 检查云函数接收参数

## 🚀 进一步解决方案

### 方案1：检查基础库版本
在 `app.json` 中确保：
```json
{
  "requiredPrivateInfos": [
    "getUserProfile"
  ],
  "lazyCodeLoading": "requiredComponents"
}
```

### 方案2：使用button获取用户信息
如果API仍有问题，改为：
```xml
<button open-type="getUserInfo" bindgetuserinfo="onGetUserInfo">
  获取用户信息
</button>
```

### 方案3：手动设置用户信息
作为临时方案，在用户中心添加"修改昵称"功能。
